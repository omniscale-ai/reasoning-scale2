# ArCHer: Training Language Model Agents via Hierarchical Multi-Turn RL

## Metadata

* **File**: `files/zhou_2024_archer-hierarchical-rl.pdf`
* **Published**: 2024
* **Authors**: Yifei Zhou 🇺🇸, Andrea Zanette 🇺🇸, Jiayi Pan 🇺🇸, Sergey Levine 🇺🇸, Aviral Kumar 🇺🇸
* **Venue**: ICML 2024
* **DOI**: `10.48550/arXiv.2402.19446`

## Abstract

A broad use case of large language models (LLMs) is in goal-directed decision-making tasks (or
"agent" tasks), where an LLM needs to not just generate completions for a given prompt, but rather
make intelligent decisions over a multi-turn interaction to accomplish a task (e.g., when
interacting with the web, using tools, or providing customer support). Reinforcement learning (RL)
provides a general paradigm to address such agent tasks, but current RL methods for LLMs largely
focus on optimizing single-turn rewards. By construction, most single-turn RL methods cannot endow
LLMs with the ability to intelligently seek information over multiple turns, perform credit
assignment, or reason about their past actions -- all of which are critical in agent tasks. This
raises the question: how can we design effective and efficient multi-turn RL algorithms for LLMs? In
this paper, we develop a framework for building multi-turn RL algorithms for fine-tuning LLMs, that
preserves the flexibility of existing single-turn RL methods for LLMs (e.g., proximal policy
optimization), while accommodating multiple turns, long horizons, and delayed rewards effectively.
To do this, our framework adopts a hierarchical RL approach and runs two RL algorithms in parallel:
a high-level off-policy value-based RL algorithm to aggregate reward over utterances, and a
low-level RL algorithm that utilizes this high-level value function to train a token policy within
each utterance or turn. Our hierarchical framework, Actor-Critic Framework with a Hierarchical
Structure (ArCHer), can also give rise to other RL methods. Empirically, we find that ArCHer
significantly improves efficiency and performance on agent tasks, attaining a sample efficiency of
about 100x over existing methods, while also improving with larger model capacity (upto the 7
billion scale that we tested on).

## Overview

ArCHer addresses a fundamental mismatch between how RL is currently applied to LLMs and how LLMs are
deployed as agents. Mainstream single-turn RL approaches such as PPO and RLHF treat each generation
as an isolated bandit problem and optimize a per-prompt reward signal. This is a poor fit for agent
settings, where decisions span many utterances, information must be acquired incrementally, and the
only meaningful reward often arrives at the end of an episode. The authors argue that existing
remedies — flattening multi-turn trajectories into one long sequence and then running token-level
PPO, or training entirely at the utterance level with a supervised filtering scheme — either suffer
from extreme variance and poor credit assignment over long horizons or sacrifice the per-token
policy expressivity that makes LLM RL effective in the first place.

The paper's central contribution is a hierarchical actor-critic framework that runs two RL
algorithms in parallel at different temporal scales. A high-level off-policy critic operates over
whole utterances and is trained with temporal-difference learning on a replay buffer; a low-level
on-policy actor operates over tokens within each utterance and is trained with policy gradient using
the high-level critic's output as the advantage signal. This decoupling allows ArCHer to combine the
sample efficiency of value-based methods with the stability and expressiveness of token-level policy
gradients. Empirically the framework reaches strong agent-task performance with roughly two orders
of magnitude fewer environment interactions than online PPO baselines and continues to improve up to
a 7B-parameter base policy.

## Architecture, Models and Methods

ArCHer is structured as a two-level actor-critic. At the **utterance (high) level**, the framework
maintains an action-value Q(s, a) and a state-value V(s) where a state is the dialogue prefix and an
action is a complete utterance generated by the LLM. The critic uses double Q-learning with two
separate {Q1, V1} and {Q2, V2} heads sharing a RoBERTa-base encoder backbone (taking the [CLS] token
followed by a linear layer), updated against delayed target networks via Polyak averaging to
stabilize bootstrapped TD targets. At the **token (low) level**, ArCHer trains the LLM policy with
REINFORCE-style policy gradient where the advantage for every token in an utterance is A = Q(s, a) -
V(s), yielding a per-utterance scalar advantage that is broadcast over the tokens of that utterance.
An optional token-level value baseline trained by supervised regression onto Monte Carlo returns is
included to reduce variance on longer utterances.

The core base policy is GPT-2 (about 100M parameters). A scaling experiment uses Mistral-7B with
LoRA fine-tuning. Critics are kept much smaller than the actors. Training proceeds in alternating
phases: collect rollouts with the current LLM policy, push transitions to a replay buffer, run TD
updates on the high-level Q/V networks against target networks, and run on-policy token-level
gradient updates on the actor with the freshly-updated advantages. The discount factor at the
utterance level is set to 1.0 in the main experiments. The replay buffer size, target-network update
rate, and number of critic updates per actor update are key hyperparameters analyzed in ablations.
All baselines (online PPO, online filtered behavior cloning, CHAI utterance-level Q-learning) start
from the same supervised fine-tuning checkpoint trained on sub-optimal demonstration data so that
comparisons isolate the contribution of multi-turn RL itself rather than initialization.

## Results

* On Twenty Questions, ArCHer reaches a return of about **-17** with **fewer than 1,000**
  environment interactions, while online token-level PPO requires more than **100,000** interactions
  to reach the same level — roughly a **100x sample-efficiency boost**.
* On WebShop, an ArCHer-trained GPT-2 (~100M params) **outperforms GPT-3.5** equipped with several
  prompting strategies on the same task, despite being orders of magnitude smaller.
* ArCHer **outperforms baselines on four out of five tasks** that require hidden-information search
  (Twenty Questions, Twenty Questions 10-word, Guess My City, WebShop), and matches the best prior
  method on the simpler Detective Game.
* Filtered behavior cloning (online filtered BC) **converges quickly but plateaus suboptimally**,
  while utterance-level CHAI improves rapidly early then converges below ArCHer's final performance.
* Scaling the actor from **GPT-2 (100M)** to **Mistral-7B** continues to improve final performance,
  demonstrating that the hierarchical framework benefits from larger backbones rather than
  saturating.
* Ablations show that **larger replay buffers improve stability** but gains plateau past a
  threshold, and that the two-level decomposition is **more robust than single-level token PPO** on
  long horizons (10+ turns).
* On harder Twenty Questions splits the **gap to PPO widens with horizon length**, confirming that
  hierarchical credit assignment is the load-bearing component, not just additional optimization
  signal.

## Innovations

### Two-Timescale Hierarchical Actor-Critic for LLMs

ArCHer decouples credit assignment over utterances from policy optimization over tokens. The
high-level off-policy TD critic learns from a replay buffer at the utterance scale; the low-level
on-policy actor learns from fresh rollouts at the token scale using the high-level critic's
advantage. Prior LLM-RL work either flattened both scales into one (token-level PPO) or collapsed
both into one (utterance-only filtered BC or Q-learning). The hierarchy is the paper's key
algorithmic move and is what enables both the 100x sample efficiency and the stability over long
horizons.

### Off-Policy Value Learning at the Utterance Level

By framing each utterance as a single composite action, ArCHer can apply standard off-policy
value-based RL machinery (replay buffers, double Q-learning, target networks, Polyak averaging) that
has been hard to apply at the token level because of the combinatorial action space. This gives
ArCHer the data-reuse benefits of off-policy RL while still allowing a token-level policy gradient
that respects the LLM's autoregressive structure.

### Algorithm-Family Generalization

The authors present ArCHer as a framework rather than a single algorithm: any high-level value
algorithm (TD, IQL, CQL) can be paired with any low-level token policy algorithm (REINFORCE, PPO,
AWR). This makes the contribution a compositional recipe for multi-turn LLM RL rather than a point
solution.

### Empirical 100x Sample-Efficiency Gain on LLM Agents

To the authors' knowledge this is the first paper to demonstrate a roughly two-orders-of-magnitude
sample-efficiency gain over online PPO on multi-turn LLM agent tasks while also matching or
exceeding the asymptotic performance of utterance-level baselines.

## Datasets

ArCHer is evaluated on five interactive multi-turn agent tasks rather than static datasets:

* **Detective Game** — interactive text fiction environment used as a relatively easy multi-turn
  benchmark; sparse end-of-episode reward.
* **Twenty Questions (157-word vocabulary)** — guess a hidden word in 20 yes/no turns; primary
  hidden-information-seeking task.
* **Twenty Questions (10-word vocabulary)** — easier subset of the above used for ablations.
* **Guess My City** — guess one of 100 cities from free-form responses; longer and harder than
  Twenty Questions.
* **WebShop** — web-shopping environment with dense [0, 1] similarity rewards comparing the selected
  product against a target description; standard agent benchmark from Yao et al. 2022.

All tasks are publicly available through their original benchmark releases. No human-subject
datasets are involved. The supervised fine-tuning data used to initialize all methods is sub-optimal
demonstration data collected from each environment.

## Main Ideas

* For multi-turn LLM agents, **token-level PPO is sample-inefficient and brittle** at long horizons
  because all credit assignment has to flow through one giant flattened sequence; splitting the
  problem into utterance-level value learning and token-level policy gradient is dramatically
  better.
* **Off-policy critics over utterances** are the right way to recycle expensive LLM rollouts in
  agent settings, and pair naturally with on-policy token-level actors. This is directly relevant
  for any project that wants to do RL fine-tuning on LLM agents under a tight rollout budget.
* The hierarchical decomposition **scales with model size** — going from 100M to 7B parameters keeps
  improving final performance, suggesting hierarchical RL is not a small-model trick.
* **Granularity of the action space matters**: treating an utterance as one decision unit at the
  high level mirrors the global / subtask / atomic hierarchy used in this project's benchmark, and
  is a concrete piece of evidence that explicit granularity boundaries help credit assignment.

## Summary

This paper develops ArCHer (Actor-Critic Framework with a Hierarchical Structure), a framework for
multi-turn reinforcement learning of language model agents. The motivation is that real LLM agent
tasks — web navigation, tool use, customer support, dialogue — involve long horizons, hidden
information that has to be acquired across turns, and rewards that arrive only at the end of an
episode, but the dominant LLM-RL recipes (PPO over flattened sequences, RLHF-style single-turn
optimization) handle none of these well. The authors argue that a hierarchical framing is a natural
fix that has been missing from the LLM-RL literature.

Methodologically, ArCHer runs two RL algorithms simultaneously at different time-scales. A
high-level off-policy critic, trained with temporal-difference learning, replay buffers, and
double-Q targets, learns Q- and V-functions over utterance-level state-action pairs. A low-level
on-policy token policy gradient (REINFORCE with optional baseline) updates the LLM itself, using the
high-level critic to produce a per-utterance advantage that is broadcast over the utterance's
tokens. This decoupling preserves the per-token expressivity that makes LLM RL work while delegating
long-horizon credit assignment to a tractable utterance-level Bellman update.

The empirical evidence is striking. On Twenty Questions, Twenty Questions 10-word, Guess My City,
and WebShop, ArCHer matches or outperforms strong baselines (online PPO, online filtered BC, CHAI)
with roughly **100x fewer environment interactions** than online PPO. A GPT-2 actor (~100M
parameters) trained with ArCHer **beats GPT-3.5 with prompting** on WebShop. Performance keeps
improving as the actor is scaled from GPT-2 to Mistral-7B, suggesting the framework composes cleanly
with model scale rather than being a small-model artifact.

For the t0017 literature survey on hierarchical agents and judges, ArCHer is a load-bearing
reference. It provides direct evidence that explicit hierarchy in the action space — utterance vs
token — yields large practical gains, mirroring the global / subtask / atomic decomposition used in
this project's annotation schema. The two-level credit-assignment story is also the right shape for
the project's scope-aware vs scope-unaware experimental conditions: ArCHer is essentially a
"scope-aware" trainer that knows which level of the hierarchy each decision lives at. The main
caveats for downstream use are that all evaluation is on relatively short-horizon text games (max
~20 turns), no SWE-bench or WorkArena-style benchmarks are tested, and the critic is small
(RoBERTa-base) which may not scale to harder reasoning agents.
