# How to Brainstorm Next Tasks

## Goal

Review project state, decide which pending suggestions deserve real tasks, and turn the chosen ones
into `not_started` task folders using [`human-brainstorm`](../../skills/human-brainstorm/SKILL.md).

## Prerequisites

* At least one completed task has written `results/suggestions.json`, or you are in the initial
  `/setup-project` brainstorm that plans first tasks from `project/description.md`
* You are ready to spend 10-15 minutes making decisions

## Steps

1. Invoke the skill — no arguments:

   ```text
   /human-brainstorm
   ```

   (In Codex: `$human-brainstorm`.)

2. The skill aggregates project state automatically: tasks, open suggestions, costs, recent
   findings. It presents the full picture, then walks you through structured rounds of discussion.

3. In each round, propose new tasks, reject or reprioritize suggestions, and review in-flight tasks.
   The skill collects all decisions, then asks for explicit confirmation before applying them.

4. After confirmation, the skill creates task folders for accepted suggestions (via
   [`create-task`](../../skills/create-task/SKILL.md)), writes corrections for rejected or
   reprioritized suggestions, produces results files, opens a PR, merges it, and refreshes the
   overview dashboard.

## Verification

After the session:

```bash
uv run python -m arf.scripts.aggregators.aggregate_tasks --status not_started --format ids
uv run python -m arf.scripts.aggregators.aggregate_suggestions --format ids
```

The first lists the newly created tasks. The second reflects the effective state after your accepts,
rejects, and adjusts.

## Pitfalls

* **Accepting every suggestion.** The point of brainstorming is to reject and defer. Running every
  suggestion generated by every completed task burns budget fast.
* **Skipping the session.** Tasks created outside brainstorm bypass the strategic review. Do it
  every time a batch of tasks completes, or on a weekly cadence.
* **Brainstorming before anything has completed outside setup.** A fresh project has no suggestions
  yet. `/setup-project` handles the special first brainstorm from the project description; after
  that, run the first task before brainstorming again.
